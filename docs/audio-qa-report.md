# 音频体验专项 QA 报告

## 本轮目标

提升浏览器普通话朗读与交互音效体验，重点处理：

1. 普通播放和慢速播放差异不明显。
2. 浏览器可能优先选到非大陆普通话声音。
3. 键盘输入音效不够短促。
4. 正确、错误、完成音效缺少可测试配置。

## 已完成调整

### 普通话朗读

- 优先选择 `zh-CN` / `zh-Hans` 普通话声音。
- 只有没有大陆普通话声音时，才回退到其他中文声音。
- 播放速度按训练粒度区分：
  - 生词：`0.96`
  - 短语：`0.88`
  - 句子 / 混合复习：`0.80`
  - 慢速：`0.52`
- 朗读参数统一放在 `speech-config.js`。

### 交互音效

- 新增 `sound-effects.js` 管理音效配置。
- 键盘音效：短促 square wave，模拟机械键盘点击感。
- 正确音效：较高频双音。
- 错误音效：低频提示音。
- 完成音效：四音阶奖励反馈。

## 当前限制

浏览器 TTS 的最终音色取决于用户系统和浏览器安装的语音包。当前版本已经优先选择大陆普通话声音，但如果系统没有可用普通话语音，浏览器仍会使用可用中文声音作为回退。

## 老板试听重点

1. 点击“播放”和“慢速”，确认慢速明显慢但不拖沓。
2. 输入拼音时确认键盘音效短促，不刺耳。
3. 答对、答错、完成时确认反馈音区分明显。
