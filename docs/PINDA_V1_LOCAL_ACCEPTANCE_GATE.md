# Pinda V1 本地总验收闸门

本文件是 Pinda V1 所有本地任务的最高优先级验收规则。

未经本地完整验收，禁止 commit、push、创建 PR、合并 main、上传 GitHub、固化标准。

## 1. 产品结构闸门

1. 学习入口只允许：生词、句子。
2. 越南语入口只允许：`Từ vựng`、`Câu`。
3. 禁止重新出现：关键词、短语、phrase 独立模块、keyword 独立模块。

## 2. 素材结构闸门

1. 素材 `item.type` 只允许：`word`、`sentence`。
2. 禁止：`phrase`、`world`、`keyword`、`vocabulary_phrase`。
3. `word` = 1～4 个汉字。
4. `word` 可以包含 1 字、2 字、3 字、4 字。
5. `word` 不能全部是 4 字固定表达。
6. `word` 必须优先包含真实高频核心词，尤其 2 字词。
7. 第一批测试素材至少包含 3 条 2 字核心词。
8. 不允许为了 TTS 稳定性，把 `word` 全部做成 4 字。
9. `sentence` = 5 个及以上汉字。

## 3. sentence.segments 闸门

每条 sentence 必须有 `segments` 数组。

`segments` 不是形式字段，必须是真正用于前端下划线拆解的自然词块数组。

硬性规则：

1. 一个 `segment` = 一个前端输入块 = 一条底部下划线。
2. 5 个及以上汉字的 `sentence` 禁止把整句作为唯一 segment。
3. 如果 `segments.length === 1`，且该 `segment.text` 去掉标点后等于整句 `chinese_text` 去掉标点后的内容，直接判为不通过。
4. `segments.map(segment.text).join("")` 去掉标点和空格后，必须等于 `chinese_text` 去掉标点和空格后的内容。
5. 固定词、自然短语、功能词块必须拆成独立 segment，不能为了省事合并成整句块。
6. 浏览器验收时，长句下划线数量必须与 `segments.length` 一致。
7. 浏览器中如果长句只出现一个整句输入框或一整条下划线，直接判为不通过。

每个 segment 必须包含：

1. `text`
2. `pinyin`
3. `vietnamese`
4. `segment_type`
5. `pos`

`segment_type` 只允许：

1. 词
2. 短语
3. 功能词块

`pos` 只允许：

1. 名词
2. 动词
3. 形容词
4. 代词
5. 副词
6. 功能词

禁止 `pos` 出现：生词、短语、关键词、word、phrase、sentence、vocab、keyword。

## 4. segment 标签闸门

句子拆解卡片底部标签必须由唯一函数生成：

```text
displaySegmentLabel(segment)
```

显示格式必须是：

```text
segment_type + " · " + pos
```

允许示例：

1. 词 · 名词
2. 短语 · 动词
3. 短语 · 名词
4. 功能词块 · 功能词

禁止显示：

1. 生词
2. 关键词
3. 单独的短语
4. 单独的名词
5. 单独的动词
6. word
7. phrase
8. sentence

如果标签不包含 ` · `，直接判为不通过。

## 5. 拼音判断闸门

拼音输入判断必须分三种状态：

1. 完整正确：绿色，并允许通过。
2. 部分正确但未完整：不红，不通过。
3. 拼音错误：红色，不通过。

要求：

1. 带声调和不带声调尽量兼容。
2. 空格不同不应直接判错。
3. 数量不完整不等于拼音错误。
4. 红色只能表示拼音本身错误。
5. 用户输入部分正确时，应给温和反馈。

## 6. 样式作用域闸门

长句词块普通状态下可以多色下划线，但必须满足：

1. 多色下划线只作用于练习页句子拆解区域。
2. 不得污染首页标题。
3. 不得污染导航栏。
4. 不得污染按钮。
5. 不得污染普通文本。
6. 正确状态仍为绿色。
7. 错误状态仍为红色。
8. 普通多色不参与判题逻辑。
9. 每个 `segment` 必须有独立底部下划线。
10. 长句普通状态不能只有一条整句下划线。

## 7. 音频绑定闸门

每条素材必须有：

1. normal 正常音频
2. slow 慢速音频

要求：

1. slow = normal 的 0.75 倍语速。
2. normal 和 slow 必须是同一句中文内容。
3. `audio_normal_path` 必须存在。
4. `audio_slow_path` 必须存在。
5. 不能生成 0KB mp3。
6. 短词 slow 必须人工听审或 ASR 辅助抽查。
7. 如果只有某条 slow 有问题，只重生成该条 slow，不重跑整包。

## 8. 真实页面路径闸门

每次 UI 错误必须先回答：

1. 页面文字来自哪个文件？
2. 页面文字来自哪个字段？
3. 哪个函数渲染？
4. 是否有旧 fallback？
5. 当前真实页面读取的是哪个数据源？
6. 测试是否覆盖真实页面路径？

## 9. 本地预览闸门

本地预览必须检查：

1. 首页能打开。
2. 高频汉语 / Daily 能进入。
3. 目标主题能进入。
4. 生词和句子两个入口正确。
5. normal / slow 音频能播放或有明确缺失提示。
6. 中文、拼音、越南语正常显示。
7. Console 无严重红色错误。
8. 人工截图问题对应场景已复核。

## 10. GitHub 固化前闸门

进入 GitHub 固化前必须确认：

1. 本地验收通过。
2. 老板人工确认通过。
3. `.env` 未进入 Git。
4. `dist/` 未进入 Git。
5. `node_modules/` 未进入 Git。
6. `.codegraph/` 未进入 Git。
7. 没有误上传素材工厂输出目录。
8. 修改范围符合本轮授权。
9. 有必要的回归测试。

## 11. 人工确认闸门

以下事项必须由老板人工确认：

1. 语义是否自然。
2. 越南语是否地道。
3. 音频 normal / slow 是否可接受。
4. 截图中曾出现的问题是否消失。
5. 是否允许进入 GitHub 固化。
