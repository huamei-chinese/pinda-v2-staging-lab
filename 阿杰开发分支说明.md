# 阿杰开发分支说明

## 当前稳定测试分支

- `v1-test-monday` 是主播周一测试版本。
- 这个分支只用于测试和问题复现。
- 不要在这个分支上继续开发新功能。

## 分支规则

### 禁止事项

- 不要修改 `v1-test-monday`
- 不要直接修改 `main`
- 不要删除课程内容
- 不要改核心数据结构
- 不要改现有学习流程
- 不要把实验性 UI 直接合进测试分支

### 推荐开发顺序

1. 先创建并使用 UI 美化分支：

```powershell
git switch main
git switch -c ajie-ui-v1
```

用途：

- UI 视觉继续优化
- 首页、课程页、高频主题页、训练页样式调整
- 移动端适配优化
- 不改课程数据结构

2. UI 稳定后，再创建后台开发分支：

```powershell
git switch main
git switch -c ajie-admin-v1
```

用途：

- 后台 V1 管理功能
- 课程管理原型
- 题目管理原型
- 后台权限和数据保存方案

## 核心保护规则

以下文件和数据要谨慎修改：

- `lesson-hsk*.js`
- `lesson-high-frequency-topics.js`
- `practice-answer-rules.js`
- `practice-rules.js`
- `lesson-runtime-rules.js`
- `academic-review-agent.js`

如果必须改，先说明原因，并在单独分支中修改。

## 开发前检查

每次开始开发前运行：

```powershell
git status --short
npm.cmd test
```

测试必须通过后再继续。

## 开发后验收

每次提交前至少检查：

- 首页能打开
- HSK lesson 能进入
- 高频汉语主题能进入
- 训练页能输入拼音
- 听写模式能播放语音
- 完成题目后 Enter / Space 才进入下一题
- 390px 手机宽度无横向滚动
- `npm.cmd test` 全部通过

## 合并建议

- `ajie-ui-v1` 只合并视觉稳定版本
- `ajie-admin-v1` 不要和 UI 美化混在同一个提交里
- 每次合并前先让老板或测试人员确认
- Monday Test Version 不作为日常开发分支使用
